**Create landing/js/validation.js** — Field-level validation module:

Following the research validation patterns exactly:

```javascript
// validation.js — Inline form validation with constructive error messages
// Validates on blur (not input) to avoid premature error display
```

1. **Validation patterns object:**
   - `email.pattern`: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
   - `email.message`: "Podaj poprawny adres email (np. jan@firma.pl)"
   - `required.test`: `(value) => value.trim().length > 0`
   - `required.message`: "To pole jest wymagane"
   - `minLength(min)`: returns `{ test: (value) => value.trim().length >= min, message: "Wpisz co najmniej ${min} znaków" }`

2. **`validateField(field)` function:**
   - Get error span via `field.getAttribute('aria-describedby')` then `getElementById`
   - Check required first, then email format, then minLength (from data-minlength attribute)
   - If invalid: set `aria-invalid="true"`, add class `input-error`, set error span text
   - If valid: set `aria-invalid="false"`, remove class `input-error`, clear error span
   - Return boolean isValid

3. **`validateAllFields(form)` function:**
   - Select all `input, textarea` in form
   - Call validateField on each
   - Return boolean (all valid)
   - If invalid, focus first field with `aria-invalid="true"`

4. **Auto-attach blur listeners:**
   - Wait for DOMContentLoaded
   - Select `#contact-form` form
   - Get all `input, textarea` within form
   - Attach blur listener to each: `field.addEventListener('blur', () => validateField(field))`

5. **Export for emailjs-form.js usage:**
   - Attach `validateField` and `validateAllFields` to window object: `window.FormValidation = { validateField, validateAllFields }`

**All error messages in Polish.** Use constructive wording per research guidelines.

**Create landing/js/emailjs-form.js** — EmailJS integration:

```javascript
// emailjs-form.js — EmailJS form submission handler
// Requires: emailjs SDK loaded, validation.js loaded before this script
```

1. **EmailJS initialization:**
   ```javascript
   (function() {
     // Initialize EmailJS with public key
     // User must replace this with their actual key from EmailJS dashboard
     emailjs.init({
       publicKey: "YOUR_PUBLIC_KEY", // TODO: Replace with your EmailJS public key
     });
   })();
   ```

2. **Form submission handler (DOMContentLoaded):**
   - Get form, status div, submit button, button-text span, button-loading span
   - On form submit:
     a. `event.preventDefault()` — FIRST LINE, critical per research
     b. Call `window.FormValidation.validateAllFields(form)`
     c. If not valid: set status to "Popraw błędy w formularzu" with class "error", return
     d. Show loading state: disable button, hide button-text, show button-loading, clear status
     e. Call `emailjs.sendForm('YOUR_SERVICE_ID', 'YOUR_TEMPLATE_ID', form)`
     f. On success: status text "Wiadomość wysłana! Odezwę się w ciągu 24 godzin.", class "success", reset form, reset button
     g. On error: status text "Nie udało się wysłać wiadomości. Spróbuj ponownie lub napisz bezpośrednio na email.", class "error", reset button
     h. Always reset button state in both success and error handlers

3. **Add clear TODO comments for all three placeholder values:**
   - `YOUR_PUBLIC_KEY` — get from EmailJS Account > General
   - `YOUR_SERVICE_ID` — get from EmailJS Email Services
   - `YOUR_TEMPLATE_ID` — get from EmailJS Email Templates

**Critical rules:**
- event.preventDefault() MUST be the very first line in submit handler
- Validation happens BEFORE EmailJS call (don't waste API calls on invalid data)
- All user-facing messages in Polish
- Button loading state prevents double-submission (disabled attribute)
- Error handling covers both validation errors and EmailJS API errors
- Code comments document what user needs to configure
